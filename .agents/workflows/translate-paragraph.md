# /translate-paragraph

마크다운 파일을 받아 한국어 번역이 없는 영어 단락을 자동으로 찾아 번역하는 워크플로우입니다.
이 워크플로우는 **Antigravity** 에이전트에 의해 실행됩니다.

## 주요 기능
1. 한국어 번역 없는 영어 단락 자동 감지
2. 기존 번역 검사 및 개선 (이미 번역이 있는 경우 품질 점검)
3. 각 단락의 한국어 번역 생성
4. 번역 품질 검사 (의미 정확성, 자연성, 마크다운 보존, 전문 용어 일관성)
5. 문제 발견 시 자동 재번역 및 검증
6. 원문의 영어는 절대 유지하며 바로 아래에 한국어 번역 삽입

## 사용법

다음과 같이 커맨드와 함께 파일이나 폴더 경로를 지정합니다:

**단일 파일 번역** (한국어 없는 모든 단락 자동 번역):
```
/translate-paragraph "markdown/7. Inductive-Types.md"
```

**폴더 전체 번역** (폴더 내 모든 파일을 번역):
```
/translate-paragraph "markdown"
```

**폴더 병렬 처리 제한** (지정된 개수만큼 병렬로 처리):
```
/translate-paragraph "markdown" 3
```
위 명령은 한 번에 3개 파일씩 병렬로 번역합니다.

## 번역 규칙

- **전문 용어**: 영어 원문 유지 (예: API, type theory, Antigravity)
- **마크다운 보존**: 코드블록, 링크(`[]()`), 강조(`**`, `_`) 등 모든 관련 문법 보존
- **스타일**: 자연스러운 한국어, 기술 및 학술적 톤 유지
- **⚠️ 중요**: 원문의 영어는 절대 삭제하면 안 되며, 한국어 번역을 원문 바로 아래에 추가합니다

## 검사 항목 (Antigravity 자체 점검 목록)

Antigravity는 번역 후 다음 항목을 스스로 검증합니다.
✅ **의미 정확성** - 원문과 번역의 의미 동일 여부
✅ **자연스러운 표현** - 한국어의 자연스러움
✅ **마크다운 보존** - 코드블록, 링크 등 문법 유지
✅ **전문 용어 일관성** - 프로젝트 전반 용어 통일

검사 중 문제 발견 시 내용을 스스로 교정(최대 2회)한 뒤 파일에 작성합니다.

## Antigravity의 작업 지침

### 단일 파일 번역
사용자가 `/translate-paragraph "파일"` 명령을 사용하면:

1. **파일 읽기**: 마크다운 파일을 읽습니다.
2. **단락 감지**: 한국어 번역이 없는 영어 단락들을 자동으로 찾습니다. (영어 단락 바로 아래에 한글 단락이 없는 경우)
3. **기존 번역 검사 및 개선**: 이미 한국어 번역이 있는 단락을 발견하면:
   - 번역 규칙 및 4가지 검사 항목으로 검증하고 문제가 있으면 자동으로 개선합니다.
4. **번역 수행**: 각 단락을 번역 규칙에 따라 한국어로 번역합니다.
5. **품질 검사 및 재번역**: 4가지 검사 항목을 스스로 체크하며, 실패 시 최대 2회까지 재번역합니다.
6. **파일 수정 (`multi_replace_file_content` / `replace_file_content` 활용)**: 검증된 번역을 원본 영어 단락 바로 아래에 정확히 추가합니다.
7. **결과 보고**: 완료 후 사용자에게 요약 보고합니다.

### 폴더 전체 번역
사용자가 `/translate-paragraph "폴더"` 또는 `/translate-paragraph "폴더" N` 명령을 사용하면:

1. **폴더 검색**: 폴더 내의 `.md` 파일들을 찾아 목록화합니다.
2. **병렬 처리 제한 설정**:
   - `N`이 없으면 모든 파일을 기본 배치 처리 단위로 처리합니다.
   - `N`이 있으면 지정된 개수만큼씩 묶어서 배치로 처리합니다. (예: 3개 처리 후 다음 3개)
3. **작업 실행**: 
   - 각 파일의 번역은 **단일 파일 번역과 동일한 번역 규칙과 4가지 품질 검사를 엄격히 적용**합니다.
4. **배치 처리**: 지정된 개수의 작업이 완료되면 다음 배치를 시작합니다.
5. **결과 보고**: 모든 파일 작업이 완료되면 최종 결과를 보고합니다.
