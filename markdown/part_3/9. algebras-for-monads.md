## Algebras for Monads

Posted by Bartosz Milewski under ,

This is part 25 of Categories for Programmers. Previously: F-Algebras. See the Table of Contents.

이것은 프로그래머를 위한 범주론의 25번째 파트입니다. 이전 글: F-대수(F-Algebras). 목차 보기.

If we interpret endofunctors as ways of defining expressions, algebras let us evaluate them and monads let us form and manipulate them. By combining algebras with monads we not only gain a lot of functionality but we can also answer a few interesting questions. One such question concerns the relation between monads and adjunctions. As we’ve seen, every adjunction defines a monad (and a comonad). The question is: Can every monad (comonad) be derived from an adjunction? The answer is positive. There is a whole family of adjunctions that generate a given monad. I’ll show you two such adjunction.

우리가 만약 자기 함자(endofunctors) 놈들을 수식(expressions) 따위를 빚어내는 어떤 조작으로 여긴다면, 그 수식들을 무자비하게 후려쳐가며 평가(evaluate)하게 해주는 건 대수(algebras)의 몫이고, 그걸 기워 조립하고 주무르도록(form and manipulate) 허락하는 건 모나드(monads)의 짓거리입니다. 이 대수와 모나드라는 두 괴물을 억지로 교배시키면 우린 그저 엄청난 기능의 폭격을 맞게 될 뿐만 아니라 덤으로 몇 가지 꽤 구미 당기는 얼빠진 질문들에 대한 해답도 주워 먹을 수 있습니다. 그딴 질문 중 하나가 바로 이 모나드와 수반(adjunctions)이라는 찝찝한 놈들 사이의 내통 관계를 캐묻는 겁니다. 앞서 우리가 두 눈깔로 목격했듯, 세상의 모든 수반은 예외 없이 모나드 하나를(그리고 덤으로 코모나드도 하나) 기어코 게워내고야 맙니다. 그럼 질문의 본질은 이렇습니다: "과연 세상 천지의 모든 모나드(혹은 코모나드) 역시 빠짐없이 어느 빌어먹을 수반 찌꺼기로부터 쥐어 짜내질 수(derived) 있는 건가?" 대답은 가증스럽게도 '그렇다(positive)'입니다. 심지어 특정 모나드 하나를 싸지르는 수반들의 거대한 일가친척(family) 무리가 버젓이 서식하고 자빠졌습니다. 나는 지금부터 당신 면전에다 그딴 역겨운 수반 녀석들 두 마리를 가져다 바칠 생각입니다.



Let’s review the definitions. A monad is an endofunctor m equipped with two natural transformations that satisfy some coherence conditions. The components of these transformations at a are:

케케묵은 정의부터 다시 후벼파 봅시다. 모나드란 건 결국 두 개의 빌어먹을 자연 변환을 훈장처럼 주렁주렁 매달고, 몇 가지 조악한 결합 조건(coherence conditions) 따위나 순순히 만족해 바치는 시시한 자기 함자 `m` 쪼가리입니다. 어떤 `a`에서 터트린 이 자연 변환들의 성분(components) 꼬라지는 대충 이러합니다:

```
ηa :: a -> m a
μa :: m (m a) -> m a
```

An algebra for the same endofunctor is a selection of a particular object — the carrier a — together with the morphism:

동일한 자기 함자에 빌붙은 대수(algebra)라는 놈은, 그저 특정한 대상 하나 — 이른바 캐리어(carrier) `a` — 를 날름 훔친 다음 이런 비루한 사상과 뒤섞어 놓은 잡탕입니다:

```
alg :: m a -> a
```

The first thing to notice is that the algebra goes in the opposite direction to ηa. The intuition is that ηa creates a trivial expression from a value of type a. The first coherence condition that makes the algebra compatible with the monad ensures that evaluating this expression using the algebra whose carrier is a gives us back the original value:

가장 먼저 눈깔에 핏발 세워 노려봐야 할 대목은 이 빌어먹을 대수란 작자가 저 `ηa`란 놈과 역주행(opposite direction)을 벌인다는 겁니다. 그 안에 웅크린 직관이란 이런 식이죠. `ηa`는 `a` 타입의 어떤 순결한 값 하나를 채가서는 아주 유치하고 하찮은(trivial) 수식 쪼가리로 포장해 버립니다. 헌데 이 대수를 모나드 따위와 스스럼없이 부대끼게 만들어줄 첫 번째 결합 조건은 이러합니다: 즉, 이 얄팍한 수식을 저 캐리어가 `a`인 평범한 대수를 써서 후려치면(평가해 버리면), 기어코 아무 흠결 없는 최초의 원판 값(original value)으로 되돌려 놓는다는 걸 맹세(ensures)하는 겁니다:

```
alg ∘ ηa = ida
```

The second condition arises from the fact that there are two ways of evaluating the doubly nested expression m (m a). We can first apply μa to flatten the expression, and then use the evaluator of the algebra; or we can apply the lifted evaluator to evaluate the inner expressions, and then apply the evaluator to the result. We’d like the two strategies to be equivalent:

두 번째 구질구질한 조건은 `m (m a)`라는 구역질 나게 겹겹이 중첩된(doubly nested) 수식을 헤집어 평가하는 데 무려 두 가지 더러운 꼼수가 도사리고 있다는 사실에서 꾸역꾸역 기어 나옵니다. 첫째는 먼저 저 무자비한 `μa`를 후려쳐 그 징그러운 중첩을 죄다 짓이겨 평평하게 만들고(flatten), 그런 다음에 대수의 평가자를 점잖게 들이미는 방식입니다. 둘째는 우선 그 멍청한 평가자를 덜미 쥐고 들어올려(lifted) 알맹이 속의 내부 수식(inner expressions)들부터 차례로 패 죽인 다음, 그 너덜너덜해진 잔해 위에다 또다시 평가자를 갈겨대는 짓거리입니다. 우리는 아주 오만하게도 이 두 얼빠진 전략이 기어이 완벽하게 동치이길(equivalent) 요구합니다:

```
alg ∘ μa = alg ∘ m alg
```

Here, m alg is the morphism resulting from lifting alg using the functor m. The following commuting diagrams describe the two conditions (I replaced m with T in anticipation of what follows):

여기서, `m alg`란 건 그 평가자 `alg` 나부랭이를 함자 `m`의 멱살을 쥐어짜 강제로 끌어올려(lifting) 얻어맞은 사상을 가리킵니다. 아래 널부러진 교환 다이어그램 쪼가리들이 지금 떠들어댄 저 두 조건을 구질구질하게 묘사해 줍니다 (나중에 벌어질 막장 사태를 예견하여, 여기서의 `m`을 내 맘대로 죄다 `T`로 둔갑시켜 놨으니 알아서들 보십시오):

We can also express these condition in Haskell:

이 망할 조건들을 또 Haskell 방언으로 역겹게 씨부려볼 수도 있습니다:

```
alg . return = id
alg . join = alg . fmap alg
```

Let’s look at a small example. An algebra for a list endofunctor consists of some type a and a function that produces an a from a list of a. We can express this function using foldr by choosing both the element type and the accumulator type to be equal to a:

어디 자잘하고 불쌍한 예시나 하나 구경해 보시죠. 어떤 리스트 자기 함자 나부랭이 위에서 굴러먹는 대수란, 대충 어떤 잡다한 타입 `a` 하나랑 저 `a`들의 리스트 구덩이에서 `a` 하나를 쥐어짜 내는 불쌍한 함수로 버무려져 있습니다. 우리는 이 멍청한 함수를 그 유명한 `foldr`의 면상으로 뽑아낼 수 있는데, 이때 그 원소(element)의 타입과 누산기(accumulator)의 타입을 몽땅 저 구질구질한 `a`와 싸잡아 통일시켜버리는 만행을 저지르면 됩니다:

```
foldr :: (a -> a -> a) -> a -> [a] -> a
```

This particular algebra is specified by a two-argument function, let’s call it f, and a value z. The list functor happens to also be a monad, with return turning a value into a singleton list. The composition of the algebra, here foldr f z, after return takes x to:

이 유별난 대수는 두 개의 인수를 게걸스레 삼키는 함수 — 이걸 만만하게 대충 `f`라고 쳐 부릅시다 — 랑, 그리고 `z`라는 찌꺼기 하나로 아주 구체적으로 적시됩니다. 이놈의 뻔뻔한 리스트 함자란 작자 역시, 어떤 가엾은 값 하나를 덜컥 주워다 알맹이 하나짜리 고독방(singleton list)에 처박는 `return`의 마법을 곁들여 기어코 모나드 노릇까지 겸하고 자빠졌죠. 이제 이 대수의 합체 덩어리, 즉 여기서의 `foldr f z`가 `return`의 등쌀을 뒤집어쓴 채 어떤 희생양 `x`를 낚아채서 질질 끌고 가는 꼬라지를 봅시다:

```
foldr f z [x] = x `f` z
```

where the action of f is written in the infix notation. The algebra is compatible with the monad if the following coherence condition is satisfied for every x:

여기서 저 `f`가 부리는 행각은 건방지게도 중위 표기법(infix notation)의 탈을 뒤집어쓰고 쓰여 있습니다. 여하튼 당신이 던져 주는 그 어떤 무고한 `x`에 대해서건 이 아래의 알량한 파트너 조건(coherence condition) 따위가 군말 없이 충족되기만 하면, 이 대수 놈은 기어코 모나드와 양립하며 서로 비벼댈 수 있는 자격을 얻게 됩니다:

```
x `f` z = x
```

If we look at f as a binary operator, this condition tells us that z is the right unit.

만약 우리가 저 잘난 `f`를 한낱 이항 연산자(binary operator) 취급한다면, 이 조건이 지껄이는 바는 그저 저 `z`라는 작자가 얌전히 우측 단일원(right unit) 노릇을 해처먹고 있다는 뻔한 헛소리에 불과합니다.

The second coherence condition operates on a list of lists. The action of join concatenates the individual lists. We can then fold the resulting list. On the other hand, we can first fold the individual lists, and then fold the resulting list. Again, if we interpret f as a binary operator, this condition tells us that this binary operation is associative. These conditions are certainly fulfilled when (a, f, z) is a monoid.

두 번째 고리타분한 결합 조건이란 건 리스트들의 징그러운 리스트 위에서 칼춤을 춥니다. 저 `join`이라는 도살자의 짓거리는 그 너절한 리스트 쪼가리들을 뭉텅이로 이어 붙이는(concatenates) 참극을 부릅니다. 그다음에야 우리는 그 처참한 결과물 리스트를 우아하게 한 번에 싹 다 폴드(fold)해서 깔아뭉갤 수 있죠. 그 반대급부로, 우린 먼저 그 알량한 하위 리스트들부터 일일이 쥐어짜서 폴드해 버린 다음, 그 잔해로 엮어낸 또 따른 리스트를 마저 폴드해 죽일 수도 있습니다. 여기서도 똑같이, 우리가 저 `f`를 빙신 같은 이항 연산자로 치부해 버리면, 이 조건이 시부리는 핵심은 결국 이 이항 연산이란 짓거리가 아주 뻔하게도 결합적(associative)이어야 한다는 명령입니다. 그리고 뻔하지만, 이건 시발 다름 아닌 무려 저 `(a, f, z)`라는 삼인방이 모노이드라는 왕관을 쓸 때나 거저 굴러들어오는 결론들이죠.

## T-algebras

## T-대수(T-algebras)

Since mathematicians prefer to call their monads T, they call algebras compatible with them T-algebras. T-algebras for a given monad T in a category C form a category called the Eilenberg-Moore category, often denoted by CT. Morphisms in that category are homomorphisms of algebras. These are the same homomorphisms we’ve seen defined for F-algebras.

수학자란 족속들이 지들의 잘난 모나드를 `T`라고 쳐 부르는 걸 애지중지하는 탓에, 그놈과 양립하는 대수들 역시 T-대수(T-algebras)라는 있어 보이는 이름으로 칭해집니다. 어떤 범주 `C` 속의 빌어먹을 모나드 `T`를 위해 결집한 T-대수들은, 아이렌베르크-무어 범주(Eilenberg-Moore category)라는 거창한 이름표를 단 또 하나의 범주를 주조해 내는데, 이게 흔히 `C^T` 따위로 역겹게 표기됩니다. 이 범주 안에서 날뛰는 사상들은 여지없이 대수의 준동형 사상(homomorphisms)들입니다. 이것들은 예전 F-대수에서 구경했던 그 썩어빠진 준동형 사상들과 토씨 하나 안 틀리고 똑같습니다.

A T-algebra is a pair consisting of a carrier object and an evaluator, (a, f). There is an obvious forgetful functor UT from CT to C, which maps (a, f) to a. It also maps a homomorphism of T-algebras to a corresponding morphism between carrier objects in C. You may remember from our discussion of adjunctions that the left adjoint to a forgetful functor is called a free functor.

T-대수란 결국 캐리어 대상과 평가자 하나가 버무려진 `(a, f)` 한 쌍에 불과합니다. 이 바닥엔 `C^T`에서 `C`로 기어 내려가는 아주 뻔한 망각 함자(forgetful functor) `U^T`가 도사리고 있는데, 이놈은 기어코 `(a, f)`를 하찮은 `a` 하나로 매핑해버립니다. 덤으로 T-대수들의 준동형 사상마저도 `C` 속 캐리어 대상들끼리의 사상으로 짓이겨 매핑해 놓죠. 어디 수반(adjunctions)을 놓고 떠들던 시절의 망령을 불러일으켜 보자면, 망각 함자 따위의 그 잘난 왼쪽 수반(left adjoint) 놈은 자유 함자(free functor)라는 고결한 이름으로 불려 왔습니다.

The left adjoint to UT is called FT. It maps an object a in C to a free algebra in CT. The carrier of this free algebra is T a. Its evaluator is a morphism from T (T a) back to T a. Since T is a monad, we can use the monadic μa (Haskell join) as the evaluator.

고로 저 `U^T`의 왼쪽 수반 놈은 `F^T`라 불립니다. 놈은 `C` 속의 불쌍한 대상 `a`를 강제로 수탈해다가 `C^T` 구역의 자유 대수(free algebra)로 매핑시켜 버리죠. 이 자유 대수의 캐리어는 놀랍게도 `T a`입니다. 그 평가자는 다름 아닌 `T (T a)`에서 다시 `T a`로 기어 돌아오는 사상이고요. 기왕지사 `T`가 버젓이 모나드 노릇을 하고 있으니, 우리는 거만하게 모나드적 `μa`(Haskell의 `join`) 놈을 평가자 대역으로 써먹어버릴 수 있습니다.

We still have to show that this is a T-algebra. For that, two coherence conditions must be satisified:

그래도 최소한 양심이 있다면 이 괴물이 T-대수란 것쯤은 손수 증명해 보여야만 합니다. 그러자면 그놈의 결합 조건 두 개가 닥치고 만족되어야만 하죠:

```
alg ∘ ηTa = idTa
```

```
alg ∘ μa = alg ∘ T alg
```

But these are just monadic laws, if you plug in μ for the algebra.

근데 저 대수 자리에다 `μ` 쪼가리를 구겨 넣어보면 알겠지만, 저건 그냥 허탈하게도 흔해 빠진 모나드 법칙(monadic laws)들의 재탕에 불과합니다.

As you may recall, every adjunction defines a monad. It turns out that the adjunction between FT and UT defines the very monad T that was used in the construction of the Eilenberg-Moore category. Since we can perform this construction for every monad, we conclude that every monad can be generated from an adjunction. Later I’ll show you that there is another adjunction that generates the same monad.

당신 뇌수리 어디쯤 아직 박혀있겠지만, 모든 수반은 예외 없이 모나드 하나를 싸지릅니다. 헌데 이 `F^T`와 `U^T`가 부대끼는 수반이 정의해 내는 부속물이란 게, 소름 돋게도 애당초 저 아이렌베르크-무어 범주를 조립할 때 제물로 바쳤던 바로 그 망할 모나드 `T` 본인이라는 참극이 까발려집니다. 우리가 이 역겨운 건축 짓거리를 어떠한 모나드를 대상으로든 똑같이 벌일 수 있는 마당이니, 고로 지상에 존재하는 그 모든 모나드는 필경 어떤 수반 덩어리로부터 생성될 수밖에 없단 음울한 결론이 도출됩니다. 내가 나중에는 아예 동일한 모나드 하나를 읊어대는 전혀 엉뚱한 또 하나의 수반 녀석도 눈앞에 들이밀어 주겠습니다.

Here’s the plan: First I’ll show you that FT is indeed the left adjoint of UT. I’ll do it by defining the unit and the counit of this adjunction and proving that the corresponding triangular identities are satisfied. Then I’ll show you that the monad generated by this adjunction is indeed our original monad.

자, 작전은 이렇습니다: 우선 첫 빠따로 저 `F^T` 놈이 실로 `U^T`의 왼쪽 수반이란 사실부터 증명해 바치겠습니다. 이 수반의 단일원(unit)과 여단일원(counit) 따위를 억지로 정의한 다음 삼각 항등식(triangular identities)인지 뭔지가 군말 없이 들어맞는단 걸 증명하는 방식 따위를 쓸 겁니다. 그런 뒤에 이 수반 찌꺼기가 싸지른 모나드가 까고 보니 우리의 그 원래 모나드가 맞다는 사실까지 마저 까발리겠습니다.

The unit of the adjunction is the natural transformation:

그 쌍방 수반의 비루한 단일원이란 건 바로 이 자연 변환입니다:

```
η :: I -> UT ∘ FT
```

Let’s calculate the a component of this transformation. The identity functor gives us a. The free functor produces the free algebra (T a, μa), and the forgetful functor reduces it to T a. Altogether we get a mapping from a to T a. We’ll simply use the unit of the monad T as the unit of this adjunction.

어디 이 빌어먹을 변환의 `a` 성분부터 후드려 패고 계산해 봅시다. 항등 함자(identity functor)는 우리에게 `a`를 토해줍니다. 뒤이어 등판한 자유 함자는 자유 대수 `(T a, μa)`를 빚어내고, 망각 함자는 그걸 또 무자비하게 껍질을 까발려 `T a` 하나로 축소시켜 버립니다. 죄다 갈아 엎고 보면 결국 남는 건 `a`에서 `T a`로 넘어가는 치졸한 매핑 하나라는 소리죠. 그저 우리는 저 모나드 `T`가 버리고 간 단일원을 주워다가 이 수반의 단일원 행세나 시키면 그만입니다.

Let’s look at the counit:

어디 이번엔 반대편 여단일원(counit)이나 좀 뜯어봅시다:

```
ε :: FT ∘ UT -> I
```

Let’s calculate its component at some T-algebra (a, f). The forgetful functor forgets the f, and the free functor produces the pair (T a, μa). So in order to define the component of the counit ε at (a, f), we need the right morphism in the Eilenberg-Moore category, or a homomorphism of T-algebras:

어느 잡다한 T-대수 `(a, f)`를 잡고 이놈의 성분이나 파헤쳐 봅시다. 멍청한 망각 함자는 기어이 저 `f`를 뇌에서 지워버리고(forgets), 자유 함자는 또 바보같이 그걸 주워다 `(T a, μa)`라는 흉물을 연성해 냅니다. 고로 저 여단일원 `ε`의 성분을 하필 `(a, f)`란 놈 위에다 억지로 꿰맞추려면, 이 망할 아이렌베르크-무어 범주 속을 관통하는 올바른 사상, 즉 T-대수들의 준동형 사상 하나가 궁해집니다:

```
(T a, μa) -> (a, f)
```

Such homomorphism should map the carrier T a to a. Let’s just resurrect the forgotten evaluator f. This time we’ll use it as a homomorphism of T-algebras. Indeed, the same commuting diagram that makes f a T-algebra may be re-interpreted to show that it’s a homomorphism of T-algebras:

이딴 변태적인 준동형 사상은 당연하게도 캐리어 `T a`를 `a`에게로 가차 없이 처박아야만 합니다. 차라리 아까 갖다 버렸던 그 평가자 `f`를 시체 닦듯 부활시켜(resurrect) 버립시다. 이번엔 그놈을 T-대수의 어엿한 준동형 사상 취급해 보는 거죠. 실로 `f`를 T-대수로 떡하게 만들어주던 바로 그 교환 다이어그램 나부랭이가, 억지로 재해석해 보면 그 자가 결국 T-대수의 준동형 사상이라는 걸 증명해 내는 꼬락서니가 될 수 있거든요:

We have thus defined the component of the counit natural transformation ε at (a, f) (an object in the category of T-algebras) to be f.

이로써 우린 저 여단일원 자연 변환 `ε`가 `(a, f)`(T-대수 범주의 불꽃 같은 대상) 위에서 작용할 그놈의 성분을, 얼렁뚱땅 `f`라고 뭉뚱그려 정의해 낸 셈입니다.

To complete the adjunction we also need to show that the unit and the counit satisfy triangular identites. These are:

이 역겨운 수반의 조각을 완성시키려면, 우린 나아가 저 단일원과 여단일원이 삼각 항등식(triangular identities)이란 걸 아주 사이좋게 만족한다는 사실까지 무자비하게 입증해 내야만 합니다. 그 거지 같은 방정식들은 이렇습니다:

The first one holds because of the unit law for the monad T. The second is just the law of the T-algebra (a, f).

첫 빠따는 모나드 `T`를 향한 단위 법칙 덕에 그저 싱겁게 맞아떨어집니다(holds). 두 번째 건 허탈하게도 T-대수 `(a, f)`의 생존 법칙 자체를 그냥 읊어댄 것뿐이고요.

We have established that the two functors form an adjunction:

그리하여 이 두 어처구니없는 함자들이 서로 몸을 섞어 하나의 수반(adjunction)을 형성해 냈단 걸 우리는 기어이 확립해 냈습니다:

```
FT ⊣ UT
```

Every adjunction gives rise to a monad. The round trip

모든 수반은 예외 없이 모나드를 싸지릅니다. 그 더러운 왕복 여행(round trip)인:

```
UT ∘ FT
```

is the endofunctor in C that gives rise to the corresponding monad. Let’s see what its action on an object a is. The free algebra created by FT is (T a, μa). The forgetful functor UT drops the evaluator. So, indeed, we have:

이 작자가 바로 `C` 구역에서 그 끔찍한 모나드를 토해내는 자기 함자에 다름없습니다. 어디 이 녀석이 불쌍한 대상 `a`를 물고 어떤 테러를 가하는지 구경이나 해봅시다. 저 `F^T`가 찍어낸 자유 대수는 앞서 말했듯 `(T a, μa)`입니다. 여기에 멍청한 망각 함자 `U^T`가 끼어들어 평가자 따윌 죄다 내다 버리니, 실로 우리 수중에 남는 거라곤:

```
UT ∘ FT = T
```

As expected, the unit of the adjunction is the unit of the monad T.

시시할 정도로 예상대로, 이 수반 나부랭이의 단일원 역시 예의 그 모나드 `T`의 단일원과 판박이로 떨어집니다.

You may remember that the counint of the adjunction produces monadic muliplication through the following formula:

당신 머갈통 어딘가엔 수반의 여단일원 놈이 예전 이런 허접한 식을 통해 모나드적 곱셈(multiplication)을 토해냈다는 잔재가 남아있을지도 모릅니다:

```
μ = R ∘ ε ∘ L
```

This is a horizontal composition of three natural transformations, two of them being identity natural transformations mapping, respectively, L to L and R to R. The one in the middle, the counit, is a natural transformation whose component at an algebra (a, f) is f.

이건 무려 세 개의 자연 변환들이 줄줄이 비엔나처럼 수평 합성(horizontal composition) 되어버린 끔찍한 덩어리입니다. 그중 양쪽 두 놈은 또 얼빠진 항등 자연 변환(identity natural transformations)이랍시고 얌전히 `L`을 `L`로, `R`을 `R`로 매핑하고 자빠졌죠. 그리고 정중앙 샌드위치 신세인 저 여단일원 녀석은, 어느 대수 `(a, f)`를 덮칠 때 그 성분이 다름 아닌 `f` 찌꺼기로 둔갑해 버리는 더러운 자연 변환인 거고요.

Let’s calculate the component μa. We first horizontally compose ε after FT, which results in the component of ε at FTa. Since FT takes a to the algebra (T a, μa), and ε picks the evaluator, we end up with μa. Horizontal composition on the left with UT doesn’t change anything, since the action of UT on morphisms is trivial. So, indeed, the μ obtained from the adjunction is the same as the μ of the original monad T.

이제 그 문제의 성분 `μa`나 좀 후들겨 패 계산해 봅시다. 우린 우선 첫 빠따로 `F^T` 뒤통수에다 `ε`를 꽂아 넣어 수평 합성시킬 건데, 그 짓거리의 말로란 결국 `F^Ta`에 찍힌 `ε`의 성분이 될 뿐입니다. 기왕지사 저 `F^T` 놈이 가엾은 `a`를 대수 `(T a, μa)`로 강제 납치하고, 속물적인 `ε`는 거기서 또 알량한 평가자나 쏙 집어 드니, 우리 손에 쥐어지는 건 결국 `μa` 뿐이라는 결론에 이릅니다. 그 왼쪽 옆구리로 `U^T`를 더 끌고 와서 수평 합성이랍시고 비벼대 봤자 세상에 변하는 건 단 하나도 없습니다. 왜냐면 `U^T`가 사상들을 두고 벌이는 그 무능한 행각이 어지간히도 맹탕거리는 짓(trivial)이니까요. 고로 까놓고 말해 저 수반 짓거리로부터 야비하게 쥐어 짜낸 `μ`는, 징그럽게도 오리지널 모나드 `T`가 물고 왔던 그 `μ`와 토씨 하나 틀리지 않고 똑같은 판박이라는 썩어빠진 진리에 봉착하게 됩니다.

## The Kleisli Category

## 클라이슬리 범주(The Kleisli Category)

We’ve seen the Kleisli category before. It’s a category constructed from another category C and a monad T. We’ll call this category CT. The objects in the Kleisli category CT are the objects of C, but the morphisms are different. A morphism fK from a to b in the Kleisli category corresponds to a morphism f from a to T b in the original category. We call this morphism a Kleisli arrow from a to b.

우린 전에 클라이슬리 범주(Kleisli category)라는 잡것을 엿본 적이 있습니다. 그건 또 다른 범주 `C`와 모나드 `T`를 징발해다 억지로 조립해 낸 범주입니다. 우린 이 끔찍한 범주를 `C_T`라고 구별해 부르겠습니다. 클라이슬리 범주 `C_T`의 대상 놈들은 원래 범주 `C`의 대상들을 그대로 베껴온 거지만, 사상들만큼은 딴판입니다. 클라이슬리 범주 내에서 `a`로부터 `b`로 향하는 어떤 사상 `fK`는, 원래의 범주 안에서 `a`로부터 `T b`로 처박히는 사상 `f`와 일맥상통합니다. 우리는 이따위 변태적인 사상을 `a`에서 `b`로 쏘는 클라이슬리 화살(Kleisli arrow)이라고 부릅니다.

Composition of morphisms in the Kleisli category is defined in terms of monadic composition of Kleisli arrows. For instance, let’s compose gK after fK. In the Kleisli category we have:

클라이슬리 범주 안에서 벌어지는 사상들 간의 합성질은 전적으로 클라이슬리 화살들의 모나드적 합성(monadic composition)에 기대어 정의됩니다. 가령 예컨대 `fK` 뒤통수에다 `gK`를 꽂아 합성한다고 쳐 봅시다. 클라이슬리 구역 안에선 일단 이런 꼬라지가 됩니다:

```
fK :: a -> b
gK :: b -> c
```

which, in the category C, corresponds to:

이것들은 원래 범주 `C`의 구역 안에선 다음과 같이 대응되죠:

```
f :: a -> T b
g :: b -> T c
```

We define the composition:

여기서 우리는 이런 구질구질한 합성을 정의합니다:

```
hK = gK ∘ fK
```

as a Kleisli arrow in C

이것을 `C` 속의 클라이슬리 화살로 치부하면 대충 이런 모양새가 됩니다:

```
h :: a -> T c
h = μ ∘ (T g) ∘ f
```

In Haskell we would write it as:

Haskell 놈들의 방언으로는 이따위로 갈겨 쓸 겁니다:

```
h = join . fmap g . f
```

There is a functor F from C to CT which acts trivially on objects. On morphims, it maps f in C to a morphism in CT by creating a Kleisli arrow that embellishes the return value of f. Given a morphism:

여기에 대상들에 대해선 눈곱만큼의 변화도 없이 수박 겉핥기(trivially)식으로만 작용하는 `C`에서 `C_T`로의 함자 `F` 하나가 있습니다. 헌데 사상들에 대해선, 이놈이 저 `C` 속의 `f`를 물어다가 `f`의 반환값 나부랭이를 허세껏 장식하는(embellishes) 클라이슬리 화살로 조작해 `C_T`의 사상으로 꽂아 넣습니다. 어떤 사상 하나가 대충 이다지도 던져졌다면:

```
f :: a -> b
```

it creates a morphism in CT with the corresponding Kleisli arrow:

이놈은 `C_T` 속에서 이런 꼴을 한 클라이슬리 화살 사상을 뚝딱 창조해 냅니다:

```
η ∘ f
```

In Haskell we’d write it as:

Haskell로 지껄이면 대충 이따위가 됩니다:

```
return . f
```

We can also define a functor G from CT back to C. It takes an object a from the Kleisli category and maps it to an object T a in C. Its action on a morphism fK corresponding to a Kleisli arrow:

거꾸로 `C_T`에서 본래의 저 `C`에게로 굴러 떨어지는 함자 `G`라는 작자도 하나 빚어낼 수 있습니다. 이놈은 클라이슬리 범주에서 불쌍한 대상 `a` 나부랭이를 낚아채, `C` 구역의 대상 `T a`라는 흉물로 매핑해버립니다. 어떤 클라이슬리 화살에 대응되는 사상 `fK`를 상대로 이 형편없는 함자가 벌이는 짓거리는:

```
f :: a -> T b
```

is a morphism in C:

결국 `C` 안에 이런 꼴사나운 사상을 남깁니다:

```
T a -> T b
```

given by first lifting f and then applying μ:

이건 우선 `f`의 멱살을 잡고 들어올린(lifting) 다음 여지없이 `μ`를 갖다 발라버린 짓거리의 결과물입니다:

```
μb ∘ T f
```

In Haskell notation this would read:

Haskell 식 표기로 이걸 읽어 재끼자면 이렇습니다:

```
G fT = join . fmap f
```

You may recognize this as the definition of monadic bind in terms of join.

당신도 슬슬 눈치챘겠지만, 이 짓거리는 그 잘난 `join`이라는 도구를 쳐 발라 모나드의 `bind`를 흉내 내어 정의해 놓은 뻔한 속임수에 불과합니다.

It’s easy to see that the two functors form an adjunction:

그리하여 이 얼빠진 두 함자들이 끈적거리며 또 하나의 수반(adjunction)을 야합해 낸다는 꼬라지를 그럭저럭 쉽게 목격할 수 있습니다:

```
F ⊣ G
```

and their composition G ∘ F reproduces the original monad T.

그리고 이 둘을 흉측하게 융합한 `G ∘ F`는 예지몽처럼 우리의 오리지널 모나드 `T`를 부활시켜(reproduces) 냅니다.

So this is the second adjunction that produces the same monad. In fact there is a whole category of adjunctions Adj(C, T) that result in the same monad T on C. The Kleisli adjunction we’ve just seen is the initial object in this category, and the Eilenberg-Moore adjunction is the terminal object.

고로 이게 바로 그 망할 똑같은 모나드를 싸지르는 두 번째 수반 되시겠습니다. 사실 그 안쪽을 후벼파 보면, `C` 구역에서 기어코 똑같은 모나드 `T`를 도출해 내고야 마는 수반 놈들의 집단, 이른바 수반들의 범주 `Adj(C, T)` 전체가 떡하니 도사리고 있습니다. 우리가 방금 눈깔로 똑똑히 발라본 이 클라이슬리 수반(Kleisli adjunction)은 그 수반들의 시궁창 같은 범주 속에서 초기 대상(initial object) 행세를 하고 있고, 저 망할 아이렌베르크-무어 수반(Eilenberg-Moore adjunction) 나부랭이는 탐욕스러운 끝 대상(terminal object) 노릇을 해처먹고 있습니다.

그리고 `ε`와 `δ`는 그 코모나드란 걸 규정짓는 두 개의 변태적인 자연 변환입니다 (Haskell 바닥에선, 저놈들의 성분이 각각 `extract`와 `duplicate`라는 천박한 이름으로 불리죠).

There is an obvious forgetful functor UW from the category of these coalgebras to C. It just forgets the coevaluation. We’ll consider its right adjoint FW.

이 코대수들의 범주에서 `C`를 향해 기어 내려가는 아주 뻔하디 뻔한 망각 함자 `U^W`가 도사리고 있습니다. 이놈은 그저 코평가(coevaluation) 따위를 뇌에서 지워버릴 뿐입니다. 우린 이제 이놈의 오른쪽 수반(right adjoint)인 `F^W`를 물고 늘어질 겁니다.

```
UW ⊣ FW
```

The right adjoint to a forgetful functor is called a cofree functor. FW generates cofree coalgebras. It assigns, to an object a in C, the coalgebra (W a, δa). The adjunction reproduces the original comonad as the composite UW ∘ FW.

어떤 망각 함자에 달라붙은 오른쪽 수반이란 놈은 코자유 함자(cofree functor)라는 역겨운 이름으로 불립니다. 이 `F^W`는 코자유 코대수(cofree coalgebras)들을 신나게 양산해 냅니다. 이놈은 `C` 구역에 널부러진 대상 `a`에다가, 다짜고짜 `(W a, δa)`라는 코대수를 덮어씌워 할당해 버리죠. 이 짓거리를 통한 수반 작태는 결국 두 놈을 합성한 `U^W ∘ F^W`의 형태로 오리지널 코모나드를 재차 게워냅니다.

Similarly, we can construct a co-Kleisli category with co-Kleisli arrows and regenerate the comonad from the corresponding adjunction.

아주 똑같은 작태로, 이쯤 되면 코-클라이슬리 화살(co-Kleisli arrows) 따위로 무장한 코-클라이슬리 범주(co-Kleisli category)도 기어코 조립해 내선, 그에 응당하는 수반으로부터 코모나드를 떡하니 예토전생(regenerate)시켜 버릴 수 있을 겁니다.

## Lenses

## 렌즈(Lenses)

Let’s go back to our discussion of lenses. A lens can be written as a coalgebra:

자, 아까 걷어찼던 그 렌즈(lenses) 썰판으로 다시 기어 돌아갑시다. 어떤 렌즈 쪼가리는 다음처럼 하나의 코대수(coalgebra)로 휘갈겨 질 수 있습니다:

```
coalgs :: a -> Store s a
```

for the functor Store s:

여기서 희생되는 건 저 `Store s`라는 함자입니다:

```
data Store s a = Store (s -> a) s
```

This coalgebra can be also expressed as a pair of functions:

이 잘난 코대수는 까놓고 말해 아래와 같은 한 쌍의 빈약한 함수 찌꺼기로도 능히 표현(expressed)되고 남습니다:

```
set :: a -> s -> a
get :: a -> s
```

(Think of a as standing for “all,” and s as a “small” part of it.) In terms of this pair, we have:

(저기서 `a`란 건 대충 덩어리 "전체(all)"를, `s`는 거기서 후벼 판 "작은(small)" 일부분을 뜻한다고 대가리 속에 욱여넣으십시오.) 이 불쌍한 한 쌍의 주둥이를 빌려 씨부리자면, 우리는 이런 걸 쥐게 됩니다:

```
coalgs a = Store (set a) (get a)
```

Here, a is a value of type a. Notice that partially applied set is a function s->a.

이 바닥에서 `a`는 `a` 타입의 어떤 구질구질한 값을 뜻합니다. 여기서 저 `set` 나부랭이가 건방지게 부분 적용(partially applied)되어 사실상 `s -> a` 꼴의 함수 노릇을 해처먹고 있다는 데 핏발 서게 주목하십시오.

We also know that Store s is a comonad:

우리는 또한 저 `Store s`가 기실 끔찍한 코모나드란 사실도 익히 꿰뚫고 있습니다:

```
instance Comonad (Store s) where
  extract (Store f s) = f s
  duplicate (Store f s) = Store (Store f) s
```

The question is: Under what conditions is a lens a coalgebra for this comonad? The first coherence condition:

그럼 질문의 핵심은 이겁니다: "도대체 어떤 망할 조건하에서 저 렌즈 나부랭이가 이 코모나드를 위한 코대수(coalgebra)로 신분 상승할 수 있단 말인가?" 그 첫 번째 결합 조건은 이러합니다:

```
εa ∘ coalg = ida
```

translates to:

이건 다음처럼 처참하게 번역됩니다:

```
set a (get a) = a
```

This is the lens law that expresses the fact that if you set a field of the structure a to its previous value, nothing changes.

이건 바로, 당신이 만약 저 `a`라는 구조체의 어떤 필드에다 다짜고짜 원래 지니고 있던 그 이전 값(previous value) 그대로 다시 세팅(set)해 버린다면 시발 세상천지에 변하는 건 아무것도 없다(nothing changes)는 뻔한 진리를 토해내는 첫 번째 렌즈 법칙(lens law)입니다.

The second condition:

그 두 번째 조건은 이렇습니다:

```
fmap coalg ∘ coalg = δa ∘ coalg
```

requires a little more work. First, recall the definition of fmap for the Store functor:

이건 쪼매 더 골머리를 썩여야만(more work) 합니다. 우선 저 `Store` 함자에 갖다 부은 `fmap`의 빌어먹을 정의부터 끄집어내 보죠:

```
fmap g (Store f s) = Store (g . f) s
```

Applying fmap coalg to the result of coalg gives us:

저 `coalg`가 싸놓은 결과물 위에다 무식하게 `fmap coalg`를 치발라 버린다면 이런 흉물이 주어집니다:

```
Store (coalg . set a) (get a)
```

On the other hand, applying duplicate to the result of coalg produces:

반대편 구석에서, 이번엔 `coalg`의 결과물에 저 저주받을 `duplicate`를 내리꽂으면 다음의 찌꺼기가 양산됩니다:

```
Store (Store (set a)) (get a)
```

For these two expressions to be equal, the two functions under Store must be equal when acting on an arbitrary s:

이 피 튀기는 두 수식이 기어이 동치(equal)가 되어 먹으려면, 저 `Store`의 밑바닥에 깔린 두 개의 멍청한 함수가 어떤 무고한 `s` 위에서 작동할 때 완벽히 똑같은 만행을 저질러야만 합니다:

```
coalg (set a s) = Store (set a) s
```

Expanding coalg, we get:

이제 저 `coalg`의 내장을 후벼 파 전개(Expanding)해 보면, 우리는 이딴 걸 얻습니다:

```
Store (set (set a s)) (get (set a s)) = Store (set a) s
```

This is equivalent to two remaining lens laws. The first one:

이 해괴한 수식은 고스란히 남아있던 나머지 두 개의 렌즈 법칙들과 소름 돋게 동치(equivalent)가 됩니다. 그 첫 번째는:

```
set (set a s) = set a
```

tells us that setting the value of a field twice is the same as setting it once. The second law:

어떤 불쌍한 필드의 값을 두 번이나 연달아 셋업(setting)하는 멍청한 짓거리는, 그냥 단 한 번 깔끔하게 후려치는 것과 티끌만큼도 다르지 않다는 걸 역설합니다. 그 두 번째 법칙은:

```
get (set a s) = s
```

tells us that getting a value of a field that was set to s gives s back.

우리가 애초에 `s`라고 어거지로 쑤셔박아 세팅해둔(set) 그 필드에서 값을 다시 구걸해(getting) 뜯어내 봐야, 결국 뱉어내는 건 보나 마나 그놈의 `s`뿐이라는 허탈한 소리입니다.

In other words, a well-behaved lens is indeed a comonad coalgebra for the Store functor.

이 역겨운 소리들을 한데 뭉뚱그려보자면, 아주 참하고 말 잘 듣는(well-behaved) 렌즈 나부랭이란 기실 저 `Store` 함자에 기생하는 코모나드 코대수(comonad coalgebra)에 다름 아니라는 결론에 안착합니다.

## Challenges

## 도전 과제(Challenges)

- What is the action of the free functor F :: C -> CT on morphisms. Hint: use the naturality condition for monadic μ.
- Define the adjunction:
UW ⊣ FW
- Prove that the above adjunction reproduces the original comonad.

- 사상(morphisms)들을 상대로 자유 함자 `F :: C -> C_T`가 작용하는 꼬라지는 대체 무엇입니까. 힌트: 저 모나드적 `μ`를 향한 자연성 조건(naturality condition)을 악랄하게 써먹으십시오.
- 다음 수반을 친히 정의해 보십시오:
  `U^W ⊣ F^W`
- 그 알량한 위 수반 나부랭이가 기어코 당신의 오리지널 코모나드를 예토전생(reproduces)시킨다는 사실을 피똥 싸며 증명해 내십시오.
